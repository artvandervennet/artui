# artui

Accessibility-first React component library. Components in `registry/` are the source of truth and are copied verbatim into consumer projects by the CLI (shadcn-style distribution) — there is no published runtime package.

## Commands

```bash
pnpm build           # turbo run build
pnpm test            # vitest via turbo
pnpm typecheck       # tsc --noEmit across workspace
pnpm lint            # biome check (ESLint in registry only)
pnpm format          # biome format --write .
pnpm dev             # turbo run dev (docs site)
pnpm changeset       # add changeset entry before release
```

Filter to one workspace: `pnpm --filter @artui/registry test`.

## Architecture

- `registry/` — source-of-truth components. Self-contained, no cross-component imports, no dependencies on `packages/`. Anything imported here ships into the consumer's repo.
- `packages/cli` (`artui`) — installs registry components into a consumer project. Built with tsup.
- `packages/mcp` — MCP server exposing registry docs to AI assistants. Built with tsup.
- `apps/docs` — Fumadocs (Next.js 16 + Tailwind v4) docs site. Prebuild copies the registry.

## Don'ts

- Don't add a runtime dependency from a registry component to anything in `packages/` or `apps/`. The CLI copies the file as-is.
- Don't edit `registry/registry.json` by hand — generated by `registry/scripts/build-registry.ts`.
- Don't introduce ESLint config outside `registry/`. Biome is the linter/formatter elsewhere.
- Never run `git commit` (or `merge`/`rebase`/`revert`/`cherry-pick`). The user always commits manually in this repo. Stage with `git add` if helpful and stop.
